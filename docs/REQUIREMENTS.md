## 1. 需求概述

### 1.1 项目背景

开发一个个人邮件智能分类汇总Agent系统，第一期聚焦于基础功能实现，支持网易126邮箱接入，完成邮件读取、存储、分类展示的核心流程。

### 1.2 第一期目标

| 目标项 | 描述 |
|--------|------|
| 核心功能 | 126邮箱邮件读取 + 基础分类 |
| 用户价值 | 自动整理邮件，快速识别重要邮件 |
| 交付标准 | 可运行的最小功能系统 |

### 1.3 范围界定

**包含：**
- 126邮箱IMAP接入
- 邮件同步存储
- 基础邮件分类（7类）
- Web端邮件列表展示
- 账户管理功能

**不包含：**
- 多邮箱支持（Gmail/Outlook等）
- 高级摘要生成
- 向量检索功能
- 移动端适配

## 2. 功能需求

### 2.1 邮箱账户管理

#### FR-001 添加邮箱账户

| 项目 | 内容 |
|------|------|
| **描述** | 用户可添加126邮箱账户 |
| **输入** | 邮箱地址、授权码（非登录密码） |
| **处理** | 验证授权码有效性，加密存储凭证 |
| **输出** | 账户添加成功/失败提示 |
| **异常** | 授权码无效、网络连接失败 |

**业务规则：**
- BR-001: 授权码必须使用AES-256-GCM加密存储
- BR-002: 不允许添加重复邮箱账户
- BR-003: 添加账户后自动测试连接

#### FR-002 删除邮箱账户

| 项目 | 内容 |
|------|------|
| **描述** | 用户可删除已添加的邮箱账户 |
| **输入** | 账户ID |
| **处理** | 删除账户及其关联的邮件数据 |
| **输出** | 删除成功/失败提示 |

**业务规则：**
- BR-004: 删除账户前需用户确认
- BR-005: 删除账户时级联删除相关邮件数据

#### FR-003 测试账户连接

| 项目 | 内容 |
|------|------|
| **描述** | 测试邮箱账户连接是否正常 |
| **输入** | 账户ID |
| **处理** | 使用存储的凭证连接IMAP服务器 |
| **输出** | 连接成功/失败 + 错误信息 |

---

### 2.2 邮件同步

#### FR-004 手动触发同步

| 项目 | 内容 |
|------|------|
| **描述** | 用户手动触发邮件同步 |
| **输入** | 账户ID（可选，不传则同步所有账户） |
| **处理** | 连接IMAP，拉取新邮件，存储到数据库 |
| **输出** | 同步任务ID，可查询进度 |
| **异常** | 连接失败、认证失败 |

**业务规则：**
- BR-006: 同步采用增量方式，只拉取上次同步后的新邮件
- BR-007: 首次同步拉取最近30天邮件（可配置）
- BR-008: 同步失败自动重试3次

#### FR-005 定时自动同步

| 项目 | 内容 |
|------|------|
| **描述** | 系统定时自动同步邮件 |
| **触发** | 按配置的时间间隔（默认5分钟） |
| **处理** | 遍历所有启用的账户执行同步 |
| **输出** | 更新同步状态时间戳 |

**配置项：**
- `email.sync_interval`: 同步间隔（分钟），默认5
- `email.initial_days`: 首次同步天数，默认30
- `email.batch_size`: 单次拉取数量，默认50

---

### 2.3 邮件存储

#### FR-006 邮件数据存储

| 字段 | 类型 | 说明 |
|------|------|------|
| message_id | VARCHAR(255) | 邮件唯一标识 |
| account_id | BIGINT | 所属账户 |
| sender_email | VARCHAR(255) | 发件人邮箱 |
| sender_name | VARCHAR(255) | 发件人名称 |
| subject | VARCHAR(512) | 邮件主题 |
| content | TEXT | 纯文本正文 |
| content_html | TEXT | HTML正文 |
| received_at | DATETIME | 接收时间 |
| category | VARCHAR(50) | 分类（默认unclassified） |
| priority | VARCHAR(20) | 优先级 |
| status | VARCHAR(20) | 状态（unread/read） |

---

### 2.4 邮件分类

#### FR-007 触发邮件分类

| 项目 | 内容 |
|------|------|
| **描述** | 对未分类邮件进行智能分类 |
| **输入** | 邮件ID（单封）或 批量触发 |
| **处理** | 调用Agent服务，使用LLM进行分类 |
| **输出** | 分类结果（类别、优先级、置信度） |

**分类类别（可配置）：**

| 类别代码 | 名称 | 说明 |
|---------|------|------|
| work_urgent | 紧急工作 | 需要立即处理 |
| work_normal | 普通工作 | 常规工作邮件 |
| personal | 个人邮件 | 私人事务 |
| subscription | 订阅邮件 | 新闻简报等 |
| notification | 系统通知 | 系统自动消息 |
| promotion | 营销推广 | 广告促销 |
| spam | 垃圾邮件 | 无价值邮件 |

**优先级：**
- critical: 紧急
- high: 高
- medium: 中
- low: 低

#### FR-008 自动分类

| 项目 | 内容 |
|------|------|
| **描述** | 邮件同步后自动触发分类 |
| **触发** | 邮件存储完成后 |
| **处理** | 异步调用分类Agent |
| **输出** | 更新邮件分类字段 |

**配置项：**
- `classification.auto_classify`: 是否自动分类，默认true
- `classification.batch_size`: 批量分类数量，默认10

## 3. 非功能需求

### 3.1 性能需求

| ID | 需求 | 指标 |
|----|------|------|
| NFR-001 | 邮件列表加载时间 | < 2秒 |
| NFR-002 | 单封邮件分类时间 | < 10秒 |
| NFR-003 | 100封邮件同步时间 | < 60秒 |
| NFR-004 | 并发用户数 | >= 10 |

### 3.2 安全需求

| ID | 需求 |
|----|------|
| NFR-005 | 邮箱授权码必须加密存储 |
| NFR-006 | API密钥不可硬编码 |
| NFR-007 | 敏感配置通过环境变量注入 |

### 3.3 可配置性需求

| ID | 需求 | 说明 |
|----|------|------|
| NFR-008 | 同步间隔可配置 | 默认5分钟 |
| NFR-009 | 首次同步天数可配置 | 默认30天 |
| NFR-010 | LLM Provider可配置 | 支持切换 |
| NFR-011 | 分类类别可配置 | 支持自定义 |

## 4. 接口规范

### 4.1 服务端 API

#### 账户管理

```
POST   /api/v1/accounts          # 添加账户
GET    /api/v1/accounts          # 账户列表
DELETE /api/v1/accounts/:id      # 删除账户
POST   /api/v1/accounts/:id/test # 测试连接
```

#### 邮件管理

```
GET    /api/v1/emails            # 邮件列表
GET    /api/v1/emails/:id        # 邮件详情
POST   /api/v1/emails/:id/classify # 分类邮件
```

#### 同步管理

```
POST   /api/v1/sync              # 触发同步
GET    /api/v1/sync/status       # 同步状态
```

### 4.2 Agent API

```
POST   /api/v1/classify          # 分类邮件
GET    /health                   # 健康检查
```
